# DevTrack 中文操作指南

适用版本：v1.0.0

DevTrack 是本地优先的项目开发监控系统。你可以把它当作个人开发工作台，也可以通过 Supabase 把项目共享给小团队成员。

## 1. 启动项目

安装依赖：

```bash
npm install
```

本机使用：

```bash
npm run dev
```

局域网/Radmin 联机使用：

```bash
npm run dev -- --host
```

默认地址：

```text
http://localhost:5173
```

生产构建：

```bash
npm run build
```

## 2. 选择工作模式

### 单人纯净流

适合不想配置 Supabase 的用户。

特点：

- 不需要注册登录。
- 不会请求 Supabase。
- 数据保存在当前浏览器 IndexedDB。
- 可以使用项目、任务、今日指挥台、专注、番茄钟、日志、分析、备份等核心功能。

注意事项：

- 清理浏览器站点数据会删除本地数据。
- 无痕模式数据可能在关闭后丢失。
- 换浏览器、换电脑、换域名后，IndexedDB 不共享。
- 重要数据请定期进入「备份恢复」创建还原点。

### 云协作流

适合团队共享项目。

特点：

- 使用 Supabase 邮箱注册/登录。
- Owner 发布共享项目。
- 成员通过邀请链接或项目共享入口加入。
- Supabase 负责数据同步、成员权限、在线状态和协作活动流。
- Realtime 正常时以事件驱动同步为主，低频兜底同步为辅。

## 3. 创建和管理项目

1. 点击侧边栏顶部项目选择器。
2. 点击「创建新项目」。
3. 输入项目名，选择颜色和截止日期。
4. 可选模板会自动生成初始任务和里程碑。

项目图标统一使用文件夹样式，并根据项目名显示缩写。

## 4. 使用任务看板

任务看板用于日常执行。

常用操作：

- 创建任务：填写标题、说明、标签、截止时间、预估工时。
- 拖动状态：待办、进行中、待审核、已完成。
- 绑定里程碑：把任务拖到里程碑区域。
- 设置依赖：在任务编辑弹窗中选择依赖任务。
- 评论和计时：在任务详情中记录讨论和实际投入。
- 批量操作：进入批量模式后统一修改状态。

被未完成依赖阻塞的任务不能直接进入进行中。

## 5. 今日指挥台和今日任务

今日指挥台用于每天开工时快速看清重点：

- 今日任务
- 风险提醒
- 阻塞任务
- 当前节奏
- 专注记录概览

今日任务页用于发布当天任务安排，可配置截止时间、提醒时间和重复规则。

## 6. 任务依赖图

进入「任务依赖」页面后：

- 每个节点代表一个任务。
- 箭头表示依赖方向。
- 可以拖拽节点。
- 可以从节点连接点拖出连线来创建依赖。
- 选中连线后按 Delete 删除依赖。
- 红色节点表示被阻塞任务。

## 7. 里程碑、时间线、日历和甘特图

- 里程碑：管理阶段目标和完成度。
- 时间线：记录项目关键事件，可设置起止日期。
- 日历：按日期查看任务、日志和事件。
- 甘特图：用时间轴查看任务和里程碑排期。
- 数据分析：查看燃尽图、CFD、任务状态拆解和趋势。

## 8. 番茄钟与专注记录

番茄钟支持：

- 工作时长、短休息、长休息配置。
- 暂停、继续、上一阶段、提前结束、下一阶段。
- 完成后自动写入专注统计。
- 浏览器通知和声音开关。

专注记录页用于查看历史专注会话。

## 9. 开发日志

开发日志支持 Markdown。

常用操作：

- 每天写一条自己的日志。
- 团队成员同一天可以各写一条日志。
- 日历格子会显示多人的心情标记。
- 支持导入 `.md` / `.json`。
- 支持导出本月日志或全部日志为 `.md` 文件。

## 10. 备份与恢复

建议在以下时机创建还原点：

- 大版本更新前。
- 发布共享项目前。
- 批量导入或恢复前。
- 修改 Supabase 配置前。

备份目录可以在设置中选择。未登录用户使用本地设置；登录用户的备份目录设置按用户隔离保存在 IndexedDB。

## 11. AI 指令中心

DevTrack 使用 DeepSeek API。

原则：

- API Key 在网页中输入。
- 不要求后端。
- 默认不保存，刷新后即丢。
- 用户可以用自己的 Key 生成任务、计划、日志或分析内容。

## 12. Supabase 团队协作

### Owner 发布共享项目

1. 切换到云协作流。
2. 在账户页登录或注册 Supabase 账号。
3. 进入团队协作页。
4. 发布当前项目为共享项目。
5. 生成邀请链接。
6. 把链接发给成员。

### 成员加入

成员可以通过两种方式加入：

1. 打开邀请链接，注册/登录后自动加入。
2. 进入系统后，在「项目共享」区域粘贴邀请链接并加入。

### 权限说明

- Owner：管理项目和成员，拥有完整权限。
- Editor：可编辑大部分协作内容。
- Viewer：只能查看。

## 13. Radmin / 局域网连接

如果使用 Radmin 虚拟局域网：

1. Owner 启动：

```bash
npm run dev -- --host
```

2. 在设置中自动检测 LAN/Radmin IP。
3. 选择类似 `26.x.x.x` 的 Radmin IPv4。
4. 生成团队连接地址。
5. 成员在同一 Radmin 网络中打开该地址。

Radmin 负责访问你的本地开发服务，Supabase 负责账号、权限和数据同步。

## 14. 同步诊断

协作出现问题时，进入协作诊断查看：

- 当前登录用户 ID
- JWT sub
- 本地用户 ID
- 当前项目 remoteProjectId
- 待同步队列数量
- 冲突数量
- 最近同步错误
- 成员在线状态

Owner 可以看到更完整的团队诊断信息。

## 15. 账号注销 Beta

账号注销仍是 Beta。建议先用测试账号验证。

可选方式：

- 使用 Supabase Edge Function 自动注销。
- 在 Supabase Dashboard 手动删除 Auth 用户。

详见：

- `docs/supabase-edge-function-account-deletion.md`
- `docs/manual-account-deletion.md`
